# M2 — Agent CLI 接入与沙箱基线（已完成）

## 范围

- 用外部 Agent CLI 替代“只走直连 LLM”的执行路径。
- 引入 `SandboxBackend` 抽象，支持本地与容器化隔离。
- 固化 auto-approve、超时、cwd、只读源码挂载等强约束。

## 主要交付

- `SandboxBackend` 抽象
- native / docker / podman 执行后端
- AgentAdapter 基础契约
- Kilo / Claude Code / OpenCode / Roo / Copilot CLI 适配层
- watchdog 超时、命令白名单、目录隔离和只读源码挂载

## 验收口径

- 与 M1 同样的 review 场景可以由 Agent CLI 驱动完成。
- 评审过程受限于隔离工作目录、命令白名单与超时守卫。
- 为后续 MCP config 物化、Agent Runtime Bundle 和更强隔离级别留出稳定扩展位。

## 对后续阶段的影响

- M5 将在此基础上补齐 Agent Runtime Bundle、原生 MCP 配置物化与更完整的 runtime 验收。
- Podman 具体部署与 rootless 说明单独沉淀在 `../../podman.md`。
- 详细设计见 `../architecture.md` 的 `3.6.3`、`3.7` 与 `3.8`。
