# M6 — 跨 VCS 能力补齐（部分完成）

## 范围

- GitHub / GitLab / Gitea / Forgejo / P4 / SVN 的触发、拉取、最小上下文与输出衔接。
- 真实平台端到端验收与本地合同层拆分。
- VCS attribution / blame / annotate 基础能力。

## 主要交付

- GitHub 生产链路已验收，覆盖 PR 分析和 issue 创建路径。
- GitLab webhook、dispatcher 与 PR review 已实现并带单元测试。
- SVN 基础 VCS adapter 已交付：`svn diff --summarize` 列变更、`svn cat -r` 最小物化文件、`svn diff --git` 供统一 diff parser 消费。
- SVN 入站触发合同层已交付：`/triggers/svn`、`translateSvnTriggerToReviewEvent` 和 `example/svn-trigger.sh`。
- P4/SVN/Git 的 `fetchExtraContext` 都支持对未物化的相关文件做最小回拉，避免把上下文缺口发布成最终报告。
- VCS attribution 基础合同已交付：`VcsAdapter.fetchAttribution`、`AttributionRequest`、`AttributionEntry`、`AttributionResult`。
- Git / P4 / SVN attribution 实现分别解析 `git blame --line-porcelain`、`p4 annotate -c` + `p4 describe -s`、`svn blame`，并以 `ok` / `partial` / `not_found` 表达 best-effort 结果。
- `aicr.try_blame` MCP 工具与 orchestrator 接线已交付：registry/server 暴露 schema，runtime bundle manifest 宣告工具，orchestrator 将 MCP state / JSON stream attribution request 转换为 `VcsAdapter.fetchAttribution` 并回灌 follow-up pass。

## 验收沉淀

- SVN trigger 的 payload 归一化、认证、异步入口和错误路径均有 `packages/server/test/svn-webhook.test.ts` 覆盖。
- SVN adapter 与 git/P4/SVN attribution 解析均有 `packages/vcs/test/*.test.ts` 覆盖。
- 归因信息不得由模型猜测；`aicr.try_blame` 只能作为只读上下文工具，通过 VCS adapter 或 provider API 重新验证。
- MCP output collector/server、runtime manifest 与 review orchestrator tests 覆盖 `aicr.try_blame` 的成功、无 adapter 支持、无效请求、MCP state 和本地 git fixture 路径。
