# M1 — Gitea + Git + 单 LLM 最小闭环（已完成）

## 范围

- 建立 webhook → ReviewEvent → Git scoped fetch → LLM → 输出发布的最小闭环。
- 接入 Gitea webhook、Git VCS adapter、统一 diff 解析与最小评审编排。
- 以一个 OpenAI 兼容模型完成最小 review 流程。
- 在 Gitea PR 上发布行级评论与 summary。

## 主要交付

- Hono webhook receiver 与 Gitea/Forgejo 基本接入
- Git VCS adapter 与最小化拉取
- unified diff 解析与行级锚点能力
- `aicr-output` 工具收集器的最小闭环
- Gitea PR review dispatcher
- CLI `serve` / `review --dry-run` 最小入口

## 验收口径

- 本地或测试环境能从 Gitea webhook 收到事件。
- 能针对 PR 产生至少一条 line comment 或 summary。
- e2e 基线跑通，形成后续里程碑的最小工作流基准。

## 对后续阶段的影响

- M2 在这个闭环之上替换为外部 Agent CLI + 沙箱执行。
- M4 继续扩展更多输出通道与模板体系。
- 详细稳定设计已整理到 `../architecture.md` 的 `3.1`、`3.2`、`3.9` 和 `3.11`。
