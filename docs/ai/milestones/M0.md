# M0 — 项目骨架总结

## 范围

- 建立 pnpm monorepo 骨架，工作区包位于 `packages/*`。
- 固定 Node、pnpm、TypeScript、ESLint、Vitest、Prettier 与 Markdown 校验基线。
- 提供 Dockerfile 骨架：Chainguard Node 多阶段镜像、`corepack` 激活 pnpm、可配置 npm 镜像源。
- 提供 CLI/help、配置/Zod 骨架、日志与观测骨架、store/schema 骨架、CI 校验链路。
- 建立仓库级 AI 维护资产骨架：`AGENTS.md`、`.agents/skills/` 与阶段总结文档。

## AI 资产投影

- 面向 agent 的常驻规则保留在 `../AGENTS.repository-baseline.md` 与根 `AGENTS.md` 的组合中，并使用按功能命名的引用与 skills。
- AI-facing 文件与 skills 不使用里程碑编号命名；里程碑编号仅保留在阶段总结文档中。
- 更新 agent 资产时，优先合并现有内容，避免重复描述同一条仓库约束。

## 验收口径

1. `aicr --help`
2. `vitest run`
3. 配置三层合并相关单元测试
4. `markdownlint-cli2` 覆盖并校验 `Plan.md`
5. 仓库级默认验证顺序保持为：`pnpm lint` → `pnpm typecheck` → `pnpm test` → `pnpm markdownlint` → `pnpm build`

## 对后续 agent 的意义

- 后续里程碑可以继续扩展功能，但当前定义的骨架约束应保持稳定，除非同步更新本文件、`AGENTS.md`、`docs/ai/AGENTS.repository-baseline.md` 与对应 skills。
- 稳定、面向大多数任务的规则保留在 `AGENTS.md`；可复用流程写入 `.agents/skills/`；详细阶段事实优先记录在本文件。
- 不把长篇 milestone 正文复制进提示词或 skill 正文，后续若需要上下文，优先链接回本文件。
