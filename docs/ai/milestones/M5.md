# M5 — 多 Agent CLI、Runtime Bundle 与 MCP 集成（基本完成）

## 范围

- 物化 agent 原生 MCP 配置与工具清单。
- 补齐 Agent Runtime Bundle manifest。
- Kilo 适配器过期标志清理与 MCP 服务器配置注入。
- Agent stdout 结构化修复与直连 LLM 兜底。
- MCP 状态文件读取与独立 MCP 服务器进程。
- Kilo Code 端到端验收。

## 主要交付

- Runtime bundle 物化 MCP 工具清单到 `manifest.json`（`@aicr/agents` `materializeRuntimeBundle`）
- 移除 Kilo 适配器过期标志（`--dangerously-skip-permissions` 已移除，`--auto` 即为 Kilo 7.x 自动审批方式）
- Runtime bundle 支持 `mcpServers` 参数，Kilo adapter 自动写入 `mcp` 配置段到 `.kilo/kilo.json`
- Orchestrator 支持 repair retry、skip 语义归一化、direct-LLM fallback
- Agent 运行后 orchestrator 读取 `.aicr-output-state.json` 并回写 collector
- `@aicr/mcp-output` 以独立子进程启动（通过 Kilo MCP 配置注入）
- Kilo Code 端到端验收通过：生产日志证实 Kilo agent 在 sandbox 内完整运行，MCP 状态文件读取、stdout 流解析、结构化输出转换均正常工作

## 验收沉淀

- 生产环境 Kilo agent 在 sandbox 内完整运行，MCP 状态文件读取正常。
- Agent stdout 无法解析时自动触发结构化修复重试，最终走直连 LLM 兜底。
- Runtime bundle 物化后，agent 工作目录包含完整的 instructions、skills、MCP 配置与 manifest。

## 当前稳定落点

- Runtime bundle 合同：`../architecture.md` §3.7
- Agent adapter 与模型翻译：`../architecture.md` §3.7、§3.7.3

- M9 依赖 M5 的 runtime bundle 与 sandbox 集成完成部署验收。
