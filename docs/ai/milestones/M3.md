# M3 — 压缩、Scrubber、Fallback、预算、Markdown 修复、Redis 队列（已完成）

## 范围

- 在大 diff、限流、密钥污染和多并发场景下补齐运行时稳定性。
- 为 review 流程加入压缩两阶段、secret scrubber、LLM fallback、预算控制和更完整队列能力。
- 把 Markdown 修复从“能用”推进到“默认输出更稳”。

## 主要交付

- PR Compression：`summarize -> review` 两阶段压缩
- `@aicr/core/secret-scrubber.ts`
- `@aicr/llm/gateway.ts` 的 fallback chain、限流重试与预算追踪
- `@aicr/core/rate-limiter.ts`
- in-memory 队列增强与 `@aicr/core/redis-queue.ts`
- `@aicr/core/queue-worker.ts`
- 更完整的 Markdown 自动修复规则
- 更强的 OpenAI-compatible / Anthropic / Gemini 兼容层

## 关键结果

- 大于 200KB 的 diff 可以在压缩后稳定进入 review。
- prompt 前与 problems 后都经过 secret scrubber，降低泄漏风险。
- 429 / Retry-After 等速率限制具备 bounded retry 与 fallback 能力。
- 多实例与多 workspace 并发由 worker + rate limiter 协同处理。
- 输出 Markdown 的结构修复更完整，减少 comment 渲染异常。

## 验收沉淀

- BullMQ 真实 Redis 集成测试已留档。
- 压缩稳定性、429 恢复/回退、Markdown 输出质量和并发去重均已纳入验收口径。
- 详细设计与当前稳定合同见 `../architecture.md` 的 `3.3`、`3.4`、`3.5`、`3.10` 与 `3.11`。

## 对后续阶段的影响

- M5 复用这套 LLM gateway、队列和预算能力承载多 Agent CLI。
- M8 的可观测性与回放会继续覆盖这些运行期机制。
